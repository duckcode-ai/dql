# Notebook Workflow

The notebook is the main working surface for ad hoc analysis and governed authoring.

## Typical Flow

1. Open the notebook with `dql notebook`
2. Browse files, blocks, or semantic assets
3. Run SQL or DQL content
4. Inspect results and charts
5. Save notebooks or convert logic into reusable blocks

## Commands

```bash
dql notebook
dql new notebook "Revenue Analysis"
```

## Use The Notebook For

- SQL exploration
- parameterized analysis
- block editing
- chart previews
- semantic browsing
